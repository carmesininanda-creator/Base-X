# Fluxa — Dev Onboarding

Ordem de leitura para um novo desenvolvedor entender o sistema em 1 dia.

## Sequência

| # | Documento | O que aprende |
|---|-----------|---------------|
| 1 | RepoMap—EstruturadoRepositório.md | GPS do repositório — onde encontrar cada coisa |
| 2 | FLUXA_ATLAS_FINAL.pdf | Domain Map — cada domínio funcional com camadas (frontend/backend/banco/integra/Syntropy) |
| 3 | FLUXA_ADR.pdf | Architecture Decision Records — por que cada decisão foi tomada |
| 4 | FLUXA_FASE1_VISAO_VS_CODIGO.pdf | Gap entre visão e implementação atual |
| 5 | FLUXA-ROTEIRO-DE-TESTE.pdf | Como validar que não quebrou nada |

## Regras

- Leia na ordem acima
- Não mexa em nada antes de ler o ADR
- Se tiver dúvida sobre "onde fica X", consulte o RepoMap primeiro
- Se tiver dúvida sobre "por que fizeram X", consulte o ADR
